# RFC-001 — LIBERO World Cup 2026 Desktop App

> **Stack:** Tauri 2 + React 18 + TypeScript 5 + Tailwind CSS 3 + SQLite (`tauri-plugin-sql`)
> **Goal:** Simple desktop betting pool app for FIFA World Cup 2026 for Libero Law Firm internal use

---

## 1. Overview

LIBERO is a desktop app (Windows/macOS/Linux) for managing a group betting pool for the 2026 FIFA World Cup. It allows:
- Managing players (add, edit, delete)
- Players submitting match score predictions
- Admin entering real match results
- Automatic point calculation per the rulebook
- Tracking the tournament bracket (group stage → R32 → R16 → QF → SF → Final)
- Players picking tournament champion and top scorer (submitted once before the first match)

All data is stored locally in SQLite — fully offline, data persists after closing the app.

## 2. Tournament: FIFA World Cup 2026

### 2.1 Format
- **48 teams**, **12 groups of 4 teams**
- Group stage: each team plays 3 matches
- Advancement to knockout: top 2 from each group + 8 best third-placed teams = **32 teams total**
- Knockout bracket: R32 (16 matches) → R16 (8) → QF (4) → SF (2) → Final (1)
- Tournament start: June 11, 2026 | Final: July 19, 2026, MetLife Stadium

## 3. Scoring Rules (per Libero Rulebook §4)

### 3.1 Group Stage
| Prediction result | Points |
|---|---|
| Exact score (e.g. predicted 2:1, actual 2:1) | **3 pts** |
| Correct winner or draw (e.g. predicted 1:0, actual 3:0) | **1 pt** |
| Wrong prediction | 0 pts |

### 3.2 Knockout Stage
Same scoring as 3.1 — based on result after 90 minutes (regular time + stoppage time).

Additionally, if a player predicted a draw in a knockout match, they may also pick who wins in extra time / penalties:
- Correct extra time / penalty winner → **+1 pt bonus**

### 3.3 Tournament Picks (submitted before the first match kick-off)
- Correct tournament champion → wins the **"best team" prize category**
- Correct top scorer → wins the **"top scorer" prize category**
- Tournament picks are a separate prize category, not summed into match points

### 3.4 Pick Deadline
Every match must be predicted before kick-off. App should display a warning when a match has already started (no hard lock in v1 — can be added later).

## 5. Database Schema

```sql
-- Players
CREATE TABLE IF NOT EXISTS players (
  id   INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE
);

-- Teams (seeded on first launch)
CREATE TABLE IF NOT EXISTS teams (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  name        TEXT NOT NULL,
  group_label TEXT NOT NULL  -- 'A' .. 'L'
);

-- Matches
CREATE TABLE IF NOT EXISTS matches (
  id                    INTEGER PRIMARY KEY AUTOINCREMENT,
  phase                 TEXT NOT NULL,     -- 'group' | 'r32' | 'r16' | 'qf' | 'sf' | 'final'
  group_label           TEXT,              -- group stage only: 'A'..'L'
  home_team_id          INTEGER REFERENCES teams(id),   -- NULL = slot not yet filled (knockout)
  away_team_id          INTEGER REFERENCES teams(id),
  home_goals            INTEGER,           -- NULL = match not yet played
  away_goals            INTEGER,
  extra_time_winner_id  INTEGER REFERENCES teams(id),   -- who won ET/penalties (knockout only)
  match_date            TEXT,              -- ISO 8601 string
  match_order           INTEGER NOT NULL   -- for sorting within a phase
);

-- Match picks (one row per player per match)
CREATE TABLE IF NOT EXISTS picks (
  id                    INTEGER PRIMARY KEY AUTOINCREMENT,
  player_id             INTEGER NOT NULL REFERENCES players(id) ON DELETE CASCADE,
  match_id              INTEGER NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
  home_goals            INTEGER NOT NULL,
  away_goals            INTEGER NOT NULL,
  extra_time_winner_id  INTEGER REFERENCES teams(id),  -- knockout draw pick bonus
  points                INTEGER NOT NULL DEFAULT 0,
  UNIQUE(player_id, match_id)
);

-- Tournament picks (champion + top scorer, one row per player)
CREATE TABLE IF NOT EXISTS tournament_picks (
  player_id          INTEGER PRIMARY KEY REFERENCES players(id) ON DELETE CASCADE,
  champion_team_id   INTEGER REFERENCES teams(id),
  top_scorer_name    TEXT
);
```

## 7. Scoring Logic (pure function)

```ts
// src/utils/scoring.ts

export function calculatePoints(
  pick: { homeGoals: number; awayGoals: number; extraTimeWinnerId?: number | null },
  result: { homeGoals: number; awayGoals: number; extraTimeWinnerId?: number | null },
  phase: Phase
): number {
  // Exact score → 3 pts
  if (pick.homeGoals === result.homeGoals && pick.awayGoals === result.awayGoals) {
    return 3;
  }

  const pickSign   = Math.sign(pick.homeGoals   - pick.awayGoals);    // -1 | 0 | 1
  const resultSign = Math.sign(result.homeGoals - result.awayGoals);

  // Correct winner / draw direction → 1 pt
  if (pickSign === resultSign) {
    let pts = 1;
    // Knockout bonus: player predicted draw AND match was a draw after 90 min
    if (phase !== 'group' && pickSign === 0 && resultSign === 0) {
      if (
        pick.extraTimeWinnerId != null &&
        pick.extraTimeWinnerId === result.extraTimeWinnerId
      ) {
        pts += 1; // correct ET/penalties winner
      }
    }
    return pts;
  }

  return 0;
}
```

After any result is saved, recalculate all picks for that match:
```ts
// Pseudocode — called from db/results.ts after INSERT/UPDATE on matches
async function recalculatePicksForMatch(db: Database, matchId: number) {
  const match = await getMatch(db, matchId);
  if (match.homeGoals === null) return; // no result yet

  const picks = await getPicksForMatch(db, matchId);
  for (const pick of picks) {
    const pts = calculatePoints(pick, match, match.phase);
    await db.execute('UPDATE picks SET points = ? WHERE id = ?', [pts, pick.id]);
  }
}
```

## 9. Screens (Tabs)

### Tab 1 — Players
- Grid of player cards (name + total points badge)
- Inline form: text input + "Add player" button
- Delete button on each card (with confirmation dialog)

### Tab 2 — Picks
- Dropdown "Select player"
- After selecting a player: matches grouped by phase (Group A–L, then R32, R16, QF, SF, Final)
- Each match row: team names + two number inputs (home : away score)
- For knockout matches where the player typed `X:X` (draw): extra dropdown "Who wins ET/penalties?"
- Tournament picks section: champion team dropdown + top scorer text input
- Save button per group/phase section (or global save — up to implementation)

### Tab 3 — Results
- All matches grouped by phase
- Each match row: team names + two number inputs for real result
- For knockout matches that ended in a draw: dropdown for ET/penalties winner
- On save → auto-recalculate all picks for that match
- Pre-filled if result already entered

### Tab 4 — Bracket
- **Group stage section:** 12 group tables (teams + W/D/L/Pts from entered results)
- **Advancement section:** admin manually marks which teams advance (top 2 per group + 8 best 3rd place) — checkboxes or a "confirm advancement" button per group
- **Knockout bracket:** visual tree of R32 → R16 → QF → SF → Final
  - Teams filled in as admin assigns advancing teams
  - Results shown inline on each match node
  - Simple CSS/Tailwind grid layout — no React Flow, no drag-and-drop

### Tab 5 — Standings
- Table: Rank | Player | Match Points | Champion ✓/✗ | Top Scorer ✓/✗
- Sorted descending by match points
- Ex aequo: same rank number for tied players
- Champion / Top Scorer columns show ✓ only after tournament ends and admin marks the actual champion/top scorer

## 10. DB Initialization Flow

```ts
// src/db/database.ts
import Database from '@tauri-apps/plugin-sql';

let _db: Database | null = null;

export async function getDb(): Promise<Database> {
  if (!_db) {
    _db = await Database.load('sqlite:libero.db');
    await runMigrations(_db);
  }
  return _db;
}

async function runMigrations(db: Database): Promise<void> {
  await db.execute(CREATE_PLAYERS_SQL);
  await db.execute(CREATE_TEAMS_SQL);
  await db.execute(CREATE_MATCHES_SQL);
  await db.execute(CREATE_PICKS_SQL);
  await db.execute(CREATE_TOURNAMENT_PICKS_SQL);
  await seedIfEmpty(db);
}

async function seedIfEmpty(db: Database): Promise<void> {
  const rows = await db.select<[{ count: number }]>('SELECT COUNT(*) as count FROM teams');
  if (rows[0].count === 0) {
    await seedTeams(db);      // insert all 48 teams
    await seedGroupMatches(db); // insert 72 group stage matches (6 per group × 12 groups)
    await seedKnockoutSlots(db); // insert 31 empty knockout match slots
  }
}
```

Knockout match slots are created with `home_team_id = NULL` and `away_team_id = NULL`.
Admin fills them in via the Bracket tab when advancing teams.

## Appendix B — Knockout Bracket Structure

```
32 teams enter the knockout stage:
  - 24 group winners/runners-up (top 2 from each of 12 groups)
  - 8 best third-placed teams

Round of 32  →  16 matches  →  16 teams remain
Round of 16  →   8 matches  →   8 teams remain
Quarterfinals →  4 matches  →   4 teams remain
Semifinals   →   2 matches  →   2 teams remain (+ 1 third-place playoff)
Final        →   1 match    →   Champion

Total knockout matches: 16 + 8 + 4 + 2 + 1 = 31
Total tournament matches: 72 + 31 + 1 (3rd place) = 104 ✓
```

Knockout slots are seeded as empty rows (`home_team_id = NULL`).
Admin assigns teams via the Bracket tab after group stage completion.
All knockout matches support ET/penalties winner field.
