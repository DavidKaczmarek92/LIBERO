    Then the player receives 3 points for the regular time result

## Unit tests — Vitest priorities (all in English)

```
calculateGroupPhasePick(pick, result): number
  ✓ exact score → 3
  ✓ correct winner, not exact → 1
  ✓ correct draw, not exact → 1
  ✓ wrong winner → 0
  ✓ picked draw, result was a win → 0
  ✓ picked win, result was a draw → 0

calculateKnockoutPhasePick(pick, result): number
  ✓ exact regular time score → 3
  ✓ correct winner, not exact → 1
  ✓ correct draw + correct knockout winner → 2
  ✓ correct draw + wrong knockout winner → 1
  ✓ correct draw + no knockout winner pick → 1
  ✓ wrong regular time result → 0, bonus not evaluated

calculateStandings(picks, results): PlayerStanding[]
  ✓ sorted descending by points
  ✓ ex aequo — same points, same rank
  ✓ player with no picks → 0 points
  ✓ standings recalculate correctly after result edit

validateTournamentPick(pick): ValidationResult
  ✓ both champion and topScorer filled → valid
  ✓ missing champion → error
  ✓ missing topScorer → error

validateImportFile(data): ValidationResult
  ✓ valid file → ok
  ✓ missing "version" field → error
  ✓ unknown version → error
  ✓ missing "tournament" field → error
  ✓ malformed JSON → error

bracketUtils.propagateWinner(matchId, winner, matches, phases): Match[]
  ✓ winner is set as homeTeam or awayTeam in the next round match
  ✓ propagation uses flowEdges override when present
  ✓ propagation falls back to auto-bracket (phase.order) when no flowEdges
  ✓ re-editing result updates the correct next-round match
  ✓ re-editing clears downstream match results that are now stale
  ✓ final match propagation does not throw

phaseUtils.validatePhases(phases): ValidationResult
  ✓ phases with duplicate order values → error
  ✓ phases with duplicate labels → error
  ✓ empty phases array → error
  ✓ single non-knockout phase (league) → valid
  ✓ phases with gaps in order (0, 2, 3) → valid (gaps allowed)

phaseUtils.generateBracketLayout(phases, matches): FlowNode[]
  ✓ knockout phases produce one column each sorted by order
  ✓ group phase produces group nodes, not bracket column
  ✓ league-only tournament produces empty bracket layout
  ✓ manual flowPosition overrides auto-calculated x/y

templateUtils.applyTemplate(template): { phases, groups }
  ✓ "wc32" template produces 6 phases and 8 empty groups
  ✓ "league" template produces 1 phase and 1 group
  ✓ "custom" template produces empty phases and groups
  ✓ applied template phases are independent copies (editing doesn't affect template)
```
